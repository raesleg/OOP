  PEDESTRIAN CROSSING + POLICE CHASE — IMPLEMENTATION SUMMARY
═══════════════════════════════════════════════════════════════════

This document highlights every file created and modified to implement
the Pedestrian Crossing mechanic and Police Chase system for Level 1.

───────────────────────────────────────────────────────────────────
1. NEW FILES CREATED (Engine Layer)
───────────────────────────────────────────────────────────────────

  a) ICommand.java
     Package:  io.github.raesleg.engine.io
     Purpose:  Engine-level interface for the Command design pattern.
               Declares execute() and undo() — fully abstract, zero
               game knowledge.
     Pattern:  Command (GoF), Interface Segregation Principle (ISP)

  b) CommandHistory.java
     Package:  io.github.raesleg.engine.io
     Purpose:  Maintains an undo stack of ICommand objects using
               ArrayDeque. Provides executeAndRecord(), undoLast(),
               size(), and clear().
     Pattern:  Command (undo stack), Dependency Inversion (depends
               only on ICommand abstraction)

───────────────────────────────────────────────────────────────────
2. NEW FILES CREATED (Game Layer — Entities)
───────────────────────────────────────────────────────────────────

  a) Pedestrian.java
     Package:  io.github.raesleg.game.entities
     Purpose:  Sprite entity (pedestrian.png, 40×40) that crosses
               the road horizontally. Has a kinematic Box2D sensor
               body for collision detection with the player car.
               Implements IExpirable for automatic EntityManager
               cleanup.
     Pattern:  Flyweight (TextureObject texture cache), Observer
               (collision events via CollisionManager → listener)

  b) StopSign.java
     Package:  io.github.raesleg.game.entities
     Purpose:  Decorative sprite entity (stopsign.png, 60×60) placed
               on the grass shoulder beside each crosswalk. Scrolls
               with the road. No physics body — purely visual cue.
               Implements IExpirable.
     Pattern:  Flyweight (TextureObject texture cache)

───────────────────────────────────────────────────────────────────
  c) PoliceCar.java
     Package:  io.github.raesleg.game.entities
     Purpose:  Chase entity (police.png, 80×140) that spawns when
               WANTED reaches 3 stars. Approaches the player from
               behind at a speed driven by aggression factor.
               The player can maintain distance by driving fast.
               Game Over when the police car reaches the player.
     Pattern:  Flyweight (TextureObject cache), Strategy (chase
               speed is data-driven via aggression parameter)

───────────────────────────────────────────────────────────────────
3. NEW FILES CREATED (Game Layer — Zones)
───────────────────────────────────────────────────────────────────

  a) CrosswalkZone.java
     Package:  io.github.raesleg.game.zone
     Purpose:  Full-road-width sensor zone (Shape subclass) that
               detects when the player drives through a pedestrian
               crossing without braking. Creates a kinematic Box2D
               sensor body. Draws semi-transparent white crosswalk
               stripes via ShapeRenderer. Tracks player overlap
               with setPlayerInside() / isPlayerInside().
     Pattern:  Observer (sensor triggers beginContact/endContact),
               Facade (PhysicsWorld creates the Box2D body)

───────────────────────────────────────────────────────────────────
4. NEW FILES CREATED (Game Layer — Rules)
───────────────────────────────────────────────────────────────────

  a) BreakRuleCommand.java
     Package:  io.github.raesleg.game.rules
     Purpose:  Concrete ICommand that increments the WANTED meter by
               calling ruleManager.recordCrosswalkViolation().
               undo() reverses the penalty via undoLastViolation().
     Pattern:  Command (execute + undo), Single Responsibility (one
               reversible action), Open/Closed (new penalty types
               become new ICommand implementations)

───────────────────────────────────────────────────────────────────

───────────────────────────────────────────────────────────────────

  a) RuleManager.java
     Package:  io.github.raesleg.game.rules
     Changes:  Added recordCrosswalkViolation() (+1 to rulesBroken)
               and undoLastViolation() (safely decrements).

  b) GameCollisionHandler.java

               • Added setCrosswalkViolationListener() setter.
               • onCollisionBegin now detects CrosswalkZone overlap

     Package:  io.github.raesleg.game.scene
     Changes:  • Added abstract hook isGameOver() — subclasses
                 define their lose condition.

               • Added collisionHandler field and protected getter
                 getCollisionHandler() so subclasses can register
                 collision listeners.
     Pattern:  Template Method (checkLevelEnd skeleton + isGameOver
               hook)

  d) Level1Scene.java
     Package:  io.github.raesleg.game.scene
     Changes:  • Added WANTED_THRESHOLD (3), CROSSING_POSITIONS
                 array, RuleManager and CommandHistory fields, lists
                 for CrosswalkZone, Pedestrian, StopSign.
               • initLevelData() now creates RuleManager +
                 CommandHistory, wires CrosswalkViolationListener

                 crosswalk zones + pedestrians + stop signs at

  e) Level2Scene.java
     Package:  io.github.raesleg.game.scene
     Changes:  Added isGameOver() returning false (placeholder for

───────────────────────────────────────────────────────────────────
6. DESIGN PATTERNS USED
───────────────────────────────────────────────────────────────────

  • Command Pattern
      ICommand → BreakRuleCommand → CommandHistory
      Each crosswalk violation is recorded as a reversible command.

  • Template Method Pattern
      BaseGameScene.checkLevelEnd()  [final skeleton]
         └─ isGameOver()            [abstract hook]
      Level1Scene returns true when police catches the player.
      Level2Scene returns false (placeholder).

  • Observer Pattern
      CollisionManager fires beginContact/endContact →
        GameCollisionHandler.onCollisionBegin() detects CrosswalkZone →
          CrosswalkViolationListener.onCrosswalkViolation() →
            Level1Scene lambda records the violation via CommandHistory.

  • Flyweight Pattern
      TextureObject caches textures for Pedestrian and StopSign.

  • Facade Pattern
      PhysicsWorld.createBody() hides raw Box2D API complexity when
      creating kinematic sensor bodies for CrosswalkZone and Pedestrian.

8. GAMEPLAY FLOW (Level 1)
───────────────────────────────────────────────────────────────────

  1. Player drives along the sunny road (scroll-based movement).
  2. At scroll offsets 5000, 15000, 28000, and 40000, a crosswalk
     with white stripes appears across the full road width, a
     pedestrian walks across, and a stop sign sits on the left
     shoulder.
  3. The pedestrian only starts crossing once the crosswalk
     scrolls into the visible screen area. The player must brake
     and wait for the pedestrian to finish crossing.
  4. If the player enters the crosswalk sensor zone while the
     pedestrian is still crossing (or hits the pedestrian), the
     GameCollisionHandler fires the CrosswalkViolationListener.
     A violation is NOT recorded if the pedestrian has already
     finished crossing.
  5. The Level1Scene lambda creates a BreakRuleCommand and passes
     it to CommandHistory.executeAndRecord(), which increments
     the WANTED meter.
  6. The dashboard displays the updated WANTED count.
  7. When WANTED >= 3, a PoliceCar spawns below the screen and
     begins chasing the player.
  8. The police car approaches at a speed driven by the
     aggression factor (more violations = faster). The player
     can maintain distance by driving at high speed.
  9. If the player keeps breaking rules, the police catches up.
 10. Game Over occurs when the police car reaches the player’s
     Y position. checkLevelEnd() transitions to StartScene.
